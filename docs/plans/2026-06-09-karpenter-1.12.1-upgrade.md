# Karpenter 1.12.1 Upgrade + Cluster Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Update the repo to Karpenter 1.12.1 and stand up both demo clusters (`k-basic-26-06-09`, `k-hl-26-06-09`) in eu-north-1 for a workshop.

**Architecture:** Three small code edits bring versions/manifests current (CRDs unchanged — API is stable). Then clusters are created step-by-step (not via the `setup-all.sh` one-shot) for per-step visibility and failure recovery, mirroring the `/setup-demo` path. Every AWS command carries an explicit `AWS_DEFAULT_REGION=eu-north-1` because the shell default is `us-east-1`.

**Tech Stack:** AWS EKS, eksctl, Helm (OCI chart), Karpenter 1.12.1, kubectl, CloudFormation, bash scripts in `scripts/`.

**Note on "tests":** This is infrastructure work, not unit-testable code. Each task's verification step is a real command with expected output — that is the test. Do not skip verification steps.
